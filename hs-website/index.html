Topic 1 - What is Git?

what & why of git
Git is like a timeline for your code. It lets you track changes, go back in time, and collaborate smoothly with others. Whether you mess up or want to try something new, Git has your back. It's a core skill every developer should feel comfortable using.
Version Control
Git is a version control system that helps you manage changes to your code over time. Think of it like a time machine for your project.
Git vs GitHub
Git is the tool while GitHub is a platform for sharing your Git repositories in the cloud.

Topic 2 - Installation & Setup

install & config
Setting up Git takes just a few minutes. Install it on your machine, and introduce yourself to Git with your name and email. These basics are important because every Git action is logged under your identity.
Installation
Download Git from git-scm.com and follow the setup wizard.
User Identity
Set your global username and email:
git config --global user.name "your-name" git config --global user.email "you@example.com"

Topic 3 - Creating or Getting a Project

start your project
Git lets you manage any folder like a smart notebook. You can create a new project with Git or clone an existing one from the web. This is your starting point for working with code efficiently and safely.
New Local Repo
Turn any folder into a Git repository:
git init
Clone from Remote
Copy a project from GitHub to your computer:
git clone https://github.com/yourname/repo.git

Topic 4 - Making & Saving Changes

save your changes
As you edit files, Git helps you stage, review, and lock in those changes with a message. This gives your project a clear history, so every improvement or fix is easy to track and undo if needed.
Status
Check what has changed:
git status
Stage
Add your changes to staging:
git add .
Commit
Save your changes with a message:
git commit -m "your message here"

Topic 5 - Branching for Features/Fixes

work on new ideas
Branches help you experiment without fear. Try new ideas, fix bugs, or build features without touching the main version. Once it works, you can bring your changes back in. It's like having a safe draft space for your code.
Create
Start a new branch:
git branch feature-login
Switch
Move to another branch:
git checkout feature-login

Topic 6 - Merging and Fixing Conflicts

bring changes together
When your work is done, merging combines changes from different branches. Git handles it smartly, but if two people edit the same thing, you’ll resolve the difference. It keeps the team moving forward together.
Merge
Combine changes from one branch into another:
git merge feature-login
Conflict Fix
If there are conflicts, Git will show them in your files. After fixing them:
git add . git commit -m "resolve conflict"

Topic 7 - GitHub & Remote Collaboration

sync with github
Remote repositories like GitHub let you share code and collaborate with others. You can push your changes, pull updates, and work as a team ~ all while keeping track of everything using Git.
Add remote
Link your project to GitHub:
git remote add origin https://github.com/yourname/repo.git
Push
Send your changes to GitHub:
git push -u origin main
Pull
Get the latest updates from GitHub:
git pull

Topic 8 - Undoing Mistakes (Reset, Revert, Restore)

fix mistakes
Made a mistake? Don’t worry. Git offers multiple ways to undo things ~ from un-staging files, undoing commits, or rolling back changes entirely. It gives you confidence to explore without breaking your code.
Unstage File
Remove a file from staging area:
git reset HEAD filename
Undo Last Commit
Undo the latest commit but keep the changes:
git reset --soft HEAD~1
Revert Commit
Create a new commit that undoes a previous one:
git revert commit_id
